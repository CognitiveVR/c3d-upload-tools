## Directory Purposes

**Repo root:**

- Purpose: hosts the bash CLI surface and the auto-generated artifacts that operators consume after uploads.
- Contains: four entry scripts, the shared utilities library, the SDK version file, gitignored generated JSON, and the gitignored `.env` family.
- Key files: `upload-scene.sh`, `upload-object.sh`, `upload-object-manifest.sh`, `list-objects.sh`, `upload-utils.sh`, `sdk-version.txt`.

**`C3DUploadTools/`:**

- Purpose: PowerShell module root, structured for direct `Import-Module ./C3DUploadTools -Force` from the repo root.
- Contains: module manifest, module loader, `Public/`, `Private/`, `Tests/`.
- Key files: `C3DUploadTools/C3DUploadTools.psd1`, `C3DUploadTools/C3DUploadTools.psm1`.

**`C3DUploadTools/Public/`:**

- Purpose: cmdlets exported via `Export-ModuleMember` and listed in `FunctionsToExport`.
- Contains: exactly one cmdlet per file, named `Verb-Noun.ps1` (PowerShell convention).
- Key files: `Upload-C3DScene.ps1`, `Upload-C3DObject.ps1`, `Upload-C3DObjectManifest.ps1`, `Get-C3DObjects.ps1`, `Test-C3DUploads.ps1`.

**`C3DUploadTools/Private/Core/`:**

- Purpose: foundational helpers loaded before anything else.
- Contains: PowerShell classes (`C3DClasses.ps1`), `.env` loader (`Import-C3DEnvironment.ps1`), module bootstrap (`Initialize-C3DModule.ps1`), error-record factory (`New-C3DErrorRecord.ps1`), structured logger (`Write-C3DLog.ps1`).

**`C3DUploadTools/Private/Validation/`:**

- Purpose: pure validation predicates with no side effects (other than throwing).
- Contains: `Test-C3DApiKey.ps1`, `Test-C3DFileSystem.ps1`, `Test-C3DUuidFormat.ps1`.

**`C3DUploadTools/Private/Api/`:**

- Purpose: every byte of HTTP I/O and URL/multipart construction.
- Contains: `Get-C3DApiUrl.ps1`, `Set-C3DRequestHeaders.ps1`, `New-C3DMultipartFormData.ps1`, `Send-C3DHttpRequest.ps1` (the `System.Net.WebClient` transport), `Invoke-C3DApiRequest.ps1` (orchestrator).

**`C3DUploadTools/Private/Utilities/`:**

- Purpose: cross-cutting helpers not specific to validation or HTTP.
- Contains: `New-C3DUploadSession.ps1`.

**`C3DUploadTools/Tests/`:**

- Purpose: ad-hoc PowerShell test scripts (not yet a Pester suite).
- Run with `pwsh -File C3DUploadTools/Tests/<test>.ps1`.

**`test-scripts/`:**

- Purpose: bash test harnesses that drive the upload scripts end-to-end against real API endpoints.
- Key file: `test-scripts/test-all.sh` — composite regression that requires a pre-existing `SCENE_ID`.

**Test asset directories (`scene-test/`, `test-scene-vancouver/`, `object-test/`, `lantern-test/`, `lantern-test-jpg/`, `icosphere-test/`, `SciFiHelmet/`):**

- Purpose: sample glTF scenes and dynamic objects used by manual tests and the test harnesses.
- Contains:
  - Scene samples: `scene.bin`, `scene.gltf`, `screenshot.png`, optional textures, and an existing `settings.json` (auto-regenerated on upload).
  - Object samples: `<name>.bin`, `<name>.gltf`, `cvr_object_thumbnail.png` (PNG only, exact name), additional PNG/JPG/JPEG/WEBP textures.
- Notes: `SciFiHelmet/` contains a `backup-*` directory written by tooling and a per-asset `README.md`.

## Key File Locations

**Entry Points (bash):**

- `upload-scene.sh`: scene upload + post-upload screenshot upload.
- `upload-object.sh`: per-object upload + local manifest accumulation.
- `upload-object-manifest.sh`: manifest POST.
- `list-objects.sh`: read-only object listing.

**Entry Points (PowerShell):**

- `C3DUploadTools/Public/Upload-C3DScene.ps1`
- `C3DUploadTools/Public/Upload-C3DObject.ps1`
- `C3DUploadTools/Public/Upload-C3DObjectManifest.ps1`
- `C3DUploadTools/Public/Get-C3DObjects.ps1`
- `C3DUploadTools/Public/Test-C3DUploads.ps1`

**Configuration:**

- `sdk-version.txt` — single line semver consumed by both implementations.
- `.env` / `.env.sample.prod` / `.env.sample.dev` — operator-supplied env (gitignored).
- `.env.example` — documented template (committed).
- `C3DUploadTools/C3DUploadTools.psd1` — module metadata and exports.
- `cspell.json` — repo-wide spell-check vocabulary.
- `.gitignore` — excludes `*object*.json`, `**/settings.json`, `.env*`, `.DS_Store`.

**Core Logic:**

- `upload-utils.sh` — every shared bash helper (logging, env, validation, HTTP parsing, error mapping, screenshot upload).
- `C3DUploadTools/Private/Core/C3DClasses.ps1` — `C3DConfiguration`, `C3DUploadRequest`, `C3DApiResponse`, `C3DUploadResult`, plus `New-C3DConfiguration` factory.
- `C3DUploadTools/Private/Api/Send-C3DHttpRequest.ps1` — the only place that actually issues HTTP requests in the PowerShell module.

**Testing:**

- `test-scripts/test-all.sh` — composite bash regression.
- `C3DUploadTools/Tests/test-module-structure.ps1` — module-load smoke test.
- `C3DUploadTools/Tests/test-scene-upload.ps1`, `test-object-upload.ps1` — workflow tests.

**Generated artifacts (repo root, gitignored):**

- `<sceneId>_object_manifest.json` — created/updated by `upload-object.sh` and `Upload-C3DObject`; consumed by `upload-object-manifest.sh` and `Upload-C3DObjectManifest`.
- `<sceneId>_object_list.json` — written by `list-objects.sh` and `Get-C3DObjects`.
- `<sceneDir>/settings.json` — regenerated/patched on every scene upload.

## Naming Conventions

**Bash files:**

- `kebab-case.sh` for all scripts (e.g., `upload-scene.sh`, `upload-object-manifest.sh`, `list-objects.sh`, `upload-utils.sh`).
- Test scripts under `test-scripts/` use `test-<area>.sh` (e.g., `test-all.sh`, `test-env-workflow.sh`).

**Bash variables and functions:**

- Globals exported by helpers: `UPPER_SNAKE_CASE` (e.g., `SCENE_ID`, `HTTP_BODY`, `HTTP_STATUS`, `SCENE_VERSION_NUMBER`).
- Helper functions: `lower_snake_case` (e.g., `load_env_file`, `validate_uuid_format`, `parse_http_response`, `handle_http_error`, `upload_screenshot`).
- CLI flags: `--lower_snake_case` (e.g., `--scene_dir`, `--object_filename`, `--dry_run`).

**PowerShell files:**

- `Verb-Noun.ps1` matching PowerShell approved verbs (e.g., `Upload-C3DScene.ps1`, `Get-C3DObjects.ps1`, `Test-C3DApiKey.ps1`, `Send-C3DHttpRequest.ps1`).
- Test scripts under `C3DUploadTools/Tests/` use lowercase `test-<area>.ps1` except for `Test-EnvWorkflow.ps1` which follows the PS Verb-Noun convention; new test scripts should prefer `Verb-Noun.ps1`.

**PowerShell identifiers:**

- Cmdlet names: `Verb-C3DNoun` — the `C3D` prefix is part of the noun.
- Parameter names: `PascalCase` (`SceneDirectory`, `SceneName`, `Environment`, `SceneId`, `DryRun`).
- Class names: `PascalCase`, `C3D` prefix (`C3DConfiguration`, `C3DUploadRequest`, `C3DApiResponse`, `C3DUploadResult`).
- Class properties: `PascalCase` (`ApiKey`, `DefaultEnvironment`, `StatusCode`, `Body`).

**Generated JSON files:**

- `<lowercase-uuid>_object_manifest.json` and `<lowercase-uuid>_object_list.json` at repo root.
- Per-scene `settings.json` inside the scene directory.

**Environment variables:**

- `C3D_DEVELOPER_API_KEY`, `C3D_SCENE_ID`, `C3D_DEFAULT_ENVIRONMENT`, `C3D_ENV` — `C3D_` prefix, `UPPER_SNAKE_CASE`.

## Where to Add New Code

**New bash upload operation:**

- Primary code: a new `*.sh` file at repo root following the `upload-*.sh` / `list-*.sh` naming.
- Required boilerplate: `#!/bin/bash`, then `SCRIPT_DIR="$(cd "$(dirname "$0")" && pwd)"`, `source "$SCRIPT_DIR/upload-utils.sh"`, `load_env_file`.
- Reuse helpers from `upload-utils.sh`; do not re-implement logging/env/validation.
- Tests: add a harness under `test-scripts/` that `cd`s to repo root and exercises the new flow.

**New shared bash helper:**

- Add the function to `upload-utils.sh`; keep it pure where possible and document required globals.
- Update all entry scripts that should call it (`upload-scene.sh`, `upload-object.sh`, `upload-object-manifest.sh`, `list-objects.sh`).

**New public PowerShell cmdlet:**

- File: `C3DUploadTools/Public/Verb-C3DNoun.ps1` (one cmdlet per file).
- Declare exports in `C3DUploadTools/C3DUploadTools.psd1` `FunctionsToExport` (the loader also auto-exports based on filenames, but the manifest is the source of truth for the Gallery).
- Use `[CmdletBinding()]`, `[Parameter(Mandatory)]`, and `[ValidateScript()]` for inputs.
- Build the request via `New-C3DConfiguration`, `Get-C3DApiUrl`, `New-C3DMultipartFormData`, `Send-C3DHttpRequest`; return a `C3DUploadResult`.
- Tests: add `C3DUploadTools/Tests/test-<feature>.ps1`.

**New internal PowerShell helper:**

- Validation helper → `C3DUploadTools/Private/Validation/Test-C3D<Thing>.ps1`.
- HTTP-related helper → `C3DUploadTools/Private/Api/<Verb>-C3D<Thing>.ps1`.
- Logging/env/error/class change → `C3DUploadTools/Private/Core/`.
- Cross-cutting helper → `C3DUploadTools/Private/Utilities/`.
- No manifest changes needed for private functions; the loader (`C3DUploadTools/C3DUploadTools.psm1`) discovers them via `Get-ChildItem -Recurse`.

**New PowerShell class or factory:**

- Add to `C3DUploadTools/Private/Core/C3DClasses.ps1` so all classes load from one file (PowerShell classes have load-order constraints).

**New test asset:**

- Create a sibling directory at repo root (e.g., `my-asset-test/`).
- For scenes: include `scene.bin`, `scene.gltf`, `screenshot.png`; optionally `settings.json` (will be regenerated).
- For objects: include `<name>.bin`, `<name>.gltf`, `cvr_object_thumbnail.png`; optionally additional `.png`/`.jpg`/`.jpeg`/`.webp` textures.

**Documentation:**

- Operator-facing docs → `README.md`.
- Claude Code guidance → `CLAUDE.md`.
- Planning artifacts → `.planning/`.

## Special Directories

**`.planning/`:**

- Purpose: GSD planning artifacts (codebase maps, phase plans).
- Generated: yes (this file).
- Committed: per project policy (usually yes for codebase docs, varies for active plans).

**`.claude/`:**

- Purpose: Claude Code local configuration.
- Generated: yes.
- Committed: typically not.

**`SciFiHelmet/backup-*/`:**

- Purpose: timestamped backups written by upload tooling when overwriting object files.
- Generated: yes.
- Committed: appears to be committed for the existing sample (`backup-2025-12-19T22-40-11-748Z`); avoid creating new ones in the working tree.

**Auto-generated, gitignored:**

- `<uuid>_object_manifest.json` and `<uuid>_object_list.json` at repo root (`.gitignore` pattern `*object*.json`).
- Per-scene `settings.json` (`.gitignore` pattern `**/settings.json`).
- `.env`, `.env.dev`, `.env.prod`, `.env.backup`, `.env.upload2.dev`, `.env.webxr*`, `.env.sample.prod`, `.env.sample.dev` (`.gitignore` pattern `.env*`).
